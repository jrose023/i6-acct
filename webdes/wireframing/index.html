Machine-Level Representation of Programs

About CSO

Computer Systems Organization is a course that covers what occurs "under the hood" of a computer system. In this particular class, we are currently discussing assembly code, the language that exists between high-level languages that humans read and binary code that the machine reads. Assembly is a textual representation of machine code that allows programmers to understand more specifically how certain HLL operations are performed "inside" the computer. This site will give a brief guide to the details of assembly instructions and how they translate to and from high-level languages, as well as how memory during program execution.

Machine Level 2: Operations and Control

  • Arithmetic and Logic Operations
    • Two operands: add, sub, imul, sal, sar, shr, xor, and, and or instructions
    • One operand: inc, dec, neg, not
    • Multiplication: unsigned imul s, d, mul s and signed imul s, d, imul s instructions
    • Divison: unsigned div s and signed idiv s instuctions
    • cqto instruction
  • Control / Processor States
  • Condition codes
    • Carry, Zero, Sign and Overflow flags
    • cmpl b, a and test b, a instructions to set flags
    • setX d instruction to set bytes
    • Conditional Jumping: jmp, je, jne, js, jns, jg, jge, jl, jle, ja, and jb instructions
    • Conditional Moves:cmovle

Machine Level 3: Loops

  • Do-While Loops: goto
  • While Loops: jump-to-middle and do-while conversion translations
  • For Loops: (init; test; update) {body} to While Loop: init; while(test) {body; update;}
  • Switch Statements: multi-case and jump tables, fall-through case, missing case

Machine Level 4: Procedures

  • Stack Frames:
    • push and pop instructions
    • Saved registers, local varables, arguments for other function calls
  • Procedure Control Flow: passing control to beginning of procedure code/to return point call and ret instructions
  • Procedute Data Flow: passing data like procedure arguments andreturn values
  • Memory Management During Execution: too few registers, use local storage
  • Registers Usage Conventions: caller saved and callee saved registers
  • Recursion: terminal case, register save, funciton call completion

Machine Level 5: Data Structures

  • Arrays: T A[L], array loops
  • Multidimensional Arrays: T A[R][C], element access by A + K(i * C +j)
  • Multilevel Arrays: Mem[ Mem[ univ + 8 * index ] + 4 * digit ]
  • Structures: representation, pointers to members is r + 4 *idx, linked lists
  • Alignment: padding for efficiency, structure alignments by multiples of largest K bytes
  • Memory Layout: stack, heap, program data, shared libraries
